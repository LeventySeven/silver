Note on architecture asymmetry: Stagehand's CLI/SDK makes its own LLM calls (act/observe/extract each run 1-2 inference calls internally with Zod schemas). Silver never calls a model — the host LLM does that reasoning; Silver only builds the grounding substrate (snapshot, refs, ID-transform) and executes deterministic verbs. So items below are graded on "did Silver port the underlying grounding/robustness mechanism," not "does Silver run the same inference calls."

## 1. Id-grounded selector contract (never let the model emit a raw selector)
**What/why:** Stagehand's `observe()`/`act()` Zod schemas constrain `elementId` to `/^\d+-\d+$/` (`packages/core/lib/inference.ts:277-313,425-461`); the LLM can only pick an ID that was present in the snapshot, never invent a CSS/XPath selector. Resolution to a real locator happens server-side, after the model responds (`observeHandler.ts:150-217`).
**Silver: HAS IT — and hardens it further.** `silver/src/perception/refmap.ts` mints `eN` refs per snapshot generation and `groundRef()` refuses to dispatch on a ref from an old generation (`ref_stale`), not just an unknown one. Stagehand's plain `elementId` regex has no generation concept — a stale-but-syntactically-valid ID from an earlier snapshot could theoretically still resolve against a `combinedXpathMap` built for a different page state; Silver's generation gate (`refmap.ts:55-71`) closes that window explicitly (documented as the "silent-wrong-click guard").

## 2. Extract's URL-hallucination guard (schema field swap + ID reinjection)
**What/why:** `transformUrlStringsToNumericIds`/`transformSchema` (`packages/core/lib/utils.ts:400-607`) replaces every `.url()` field with an ID-pattern string before the LLM sees the schema; `injectUrls()` (`utils.ts:616-661`) splices the real URL back in from a `combinedUrlMap` after inference. The model is structurally unable to emit a fabricated URL for those fields.
**Silver: HAS IT — and closes Stagehand's own noted anti-pattern.** `silver/src/extract/transform.ts` (`transformSchema`, `ID_PATTERN = '^\\d+-\\d+$'`) is an explicit, credited port of this pattern for plain JSON Schema (keyless — no Zod dependency). `silver/src/extract/resolve.ts` is the reinjection side, and it fixes the exact weakness Stagehand's own digest flags as an anti-pattern: Stagehand's `injectUrls` silently falls back to `idToUrlMapping[id] ?? ""` on an unresolvable ID (a hallucinated ID becomes an invisible empty string); Silver's `resolveIds()` instead returns a named warning listing every unresolved ID and nulls only those fields (`resolve.ts:62-70`), and additionally gates the whole resolution on a `bundleGeneration === currentGeneration` check that Stagehand has no equivalent of.

## 3. Hybrid AX-tree pruning + text-node dedup (token-cheap, no raw HTML)
**What/why:** Stagehand never feeds raw HTML to the model — it prunes `generic`/`none`/`inlinetextbox` roles with no name/children, collapses single-child structural wrappers, and strips StaticText children that duplicate the parent's accessible name to avoid "Sign in Sign in" (`a11yTree.ts:160-224,262-278`).
**Silver: HAS IT.** `silver/src/perception/walk.ts` builds the same hybrid CDP AX+DOM tree (plus a cursor-interactive cascade Stagehand doesn't have, for `<div onclick>` pseudo-buttons — `walk.ts:12-27`) and `silver/src/perception/serialize.ts` does the equivalent structural cleanup at render time: `RootWebArea`/`WebArea` skip, empty-`generic`-wrapper collapse, empty-StaticText drop (`serialize.ts:165-171`), plus a `mergeTextLeaves()` that folds/dedups StaticText leaves into the parent's display name (`serialize.ts:209-231`) — functionally the same anti-duplication behavior Stagehand's pruning targets.

## 4. GAP — Self-healing deterministic action cache (ActCache)
**What/why:** `packages/core/lib/v3/cache/ActCache.ts` caches successful `act()` resolutions keyed by `sha256({instruction, normalizedUrl, sortedVariableKeys})` (`ActCache.ts:188-199`), replays them with **zero LLM calls** on a cache hit (`takeDeterministicAction`), and self-heals by diffing the replayed action against a fresh re-resolution — silently rewriting the cache file when the DOM has drifted rather than erroring (`ActCache.ts:264-360`). This is Stagehand's answer to "don't pay a model call for the same click every single run."
**Silver: no equivalent.** `silver/src/task/store.ts` persists a run's plan/log/checkpoints for crash-recovery/replay-as-artifact (a different thing: it's "the script IS the artifact," not a keyed selector cache), and `silver/src/memory/store.ts` is grep-first episodic markdown with no structured selector state at all. Every repeat run of the same flow (e.g. a `task` re-run, or a scripted `subagent` loop hitting the same page) currently pays a full `snapshot` + host-reasoning round-trip for every ref, even when the exact same instruction against the exact same normalized URL succeeded moments ago. Adopting a Silver-side action cache — `sha256(instruction+url)` → last-good `{role,name,nth,backendNodeId}` resolution hint, replayed via the existing `groundRef`/`toLocator` path and silently refreshed on drift — would cut repeat-run latency without adding any model call (stays keyless), since Silver already owns the exact refmap/backendNodeId primitives Stagehand's cache stores.

## 5. GAP — Reliability hardening before/during snapshot capture (network-quiet wait + DOM-depth retry ladder)
**What/why:** Two related Stagehand robustness mechanisms Silver's `snapshotNodes()` lacks:
- `waitForDomNetworkQuiet()` (`packages/core/lib/v3/handlers/handlerUtils/actHandlerUtils.ts:528-684`) tracks in-flight `Network.requestWillBeSent` counts with a stalled-request sweep and declares "quiet" 500ms after in-flight hits zero, run automatically before every `act()` snapshot — explicitly built instead of Playwright's own `networkidle` wait, which Playwright itself deprecated as flaky.
- `DOM_DEPTH_ATTEMPTS = [-1,256,128,64,32,16,8,4,2,1]` (`STAGEHAND_321_FULL_EXTRACTION.md:1317-1322`): snapshot capture starts unlimited-depth and steps down through a fixed ladder on CDP timeout/OOM for pathological deeply-nested pages, trading completeness for reliability instead of hard-failing.
**Silver: neither exists today.** `silver/src/perception/walk.ts:173` calls `cdp.send('DOM.getDocument', { depth: -1, pierce: true })` once, with no depth-fallback ladder — a pathological page that times out this single unlimited call fails the snapshot outright rather than degrading gracefully. And `silver/src/actuation/wait.ts` exposes `networkidle` only as a caller-opt-in `{load}` spec (`wait.ts:32`) — nothing in the snapshot/act path automatically waits for network quiescence, so a caller that snapshots immediately after a click that kicks off async content can race a still-loading DOM. Both are cheap, self-contained additions (no model call, no new dependency) that would close a real robustness gap Stagehand explicitly engineered for.
